Tokyo Airbnb (2022) Data Analysis using SQL
Intro
In Nov. 2022, I visited Tokyo, Japan after 7 years! The trip inspired this dashboard. My first ever trip to Japan was in 2015. It was my very first solo travel ever. Back then I was able to secure various Airbnbs for ~$55-$75 per night in various districts of Tokyo & later also Osaka. After I was impacted by workforce reduction at Domino Data Lab in early November, I decided to do a last minute trip to Tokyo. Borders reopened for tourists in October, initially for travelers traveling w/in tour groups only and later for all in the US and certain Asian countries like Hong Kong and Taiwan.
Questions used to analyze data
  • 1) What are the top five most expensive neighborhoods in Tokyo?
  • 2) What are the most popular property types?
  • 3) What are most popular listings?
  • 4) What are top ten 10 neighborhoods that have the most superhosts?

    Detailed Listings data is pulled from Inside Airbnb by searching for "Tokyo." http://insideairbnb.com/get-the-data Listings: Includes data on all the details related to lissting and host info, consisting of 75 columns and 9,787 rows.
    Analysis
    Issues converting Excel into SQL to load into SQL server:
    Excel has issues with CSV files that contains international language data. So the problem was occuring during the importing of the CSV file because by default, it doesn't support UTF-8 data. https://www.isurvey.soton.ac.uk/help/opening-csv-files-containing-international-language-data/index.html I was able to find a workaround after some research. Opened Excel and then loaded the data in UTF-8 and comma format. This worked! Next in order to proceed to converting CSV to SQL, I downloaded Notepad++ and added the "CSV Lint" plug-in for Notepad++ via https://github.com/BdR76/CSVLint/. I was able to successfully convert the CSV to SQL.
    SSMS: SSMS 2022 has an error importing data so after some digging, I was able to uninstall SSMS 2022 to SSMS 2019 and was able to import the SQL file. Next, I encountered an error when trying to query from the airbnb database despite a few refreshes. After restarting SSMS, I was able to query the data.
